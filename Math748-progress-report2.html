1.             Introduction 

Music is a beautiful language of human beings. In the long history of mankind, music has gradually formed various styles and genres. Bach is generally regarded as one of the greatest composers. Can we have more music which has the same styles and genres from Bachs music? Or can we have more songs from the same music Genres and styles. In this project, it aims to solve this problem. The training dataset is from Bachs chorales. This project will try to make models to generate midi songs from the same Bachs style.

This project is supervised learning. It will organize the data into sequences that the model can learn from. Each sequence could represent a segment of a chorale and the model will calculate the probabilities of all the possibility notes and select the highest probability note and generate the next segment. So, it is a classification problem.

The data is in lisp extension which give some challenge to read the data. And it also needs to preproduce the data to sequences pairs. Time-series characteristics also give me challenge. About the final midi generate, because the training data has only single-line melodies, the output midi has only single-line. (The data set only has the highest note if its a chord.) There is no bass chord section, so the effect is limited.

 

 

2.             Explanation of Data 

The data is from UCI database, the download link is as below:

https://archive.ics.uci.edu/dataset/25/bach+chorales

The dataset is univariate and time-series. Single-line melodies of 100 Bach chorales (originally 4 voices). It contains both categorical and integer data type. About the sources:

Chorales are from a book: (Mainous and Ottman edition. Mainous, Frank D. and Robert W. Ottman, eds. 1966.) Book name: The 371 Bach Chorales. Holt, Rinehart and Winston, New York. Number of Instances: 100 Chorales, each with around 45 events (different length). Number of Attributes: 6 (nominal) per event. They are: start-time, pitch, duration, key signature, time signature, and fermata.

 

Prepare the data for analysis: The data is the soprano line, and originally in Lisp-readable form. A python code was written to transfer lisp form to text form in one file. After transferred lisp data into txt data by shifting and adding notes to the same row, later this project used three previous notes to predict the following notes (the 4th notes) one by one. After added columns, there are 28 columns (7 features * 4 continuous notes). NA data was deleted after combining 28 columns. Because the last note doesnt have the next 3 notes. It could generate columns with NA.

 

After transferred lisp to txt file, python code was used to read and explore the data.

3.2 LDA

Linear Discriminant Analysis (LDA) is a dimensionality reduction and classification technique commonly used in statistics and machine learning. It is particularly useful for finding the linear combinations of features that best separate two or more classes in a supervised setting. LDA assumes that the data follow a normal distribution and that the classes have the same covariance matrix.

3.3 Logistic regression

Logistic regression is for binary classification problems. Here to predict more than 2 classes of Y. It can extend to handle multi-class classification. The One-vs-Rest (OvR) is a method to achieve multi-class problem. The method is: Create K binary classifiers, where K is the number of classes. Train each classifier to distinguish between one class and the rest of the classes.

For prediction, choose the class with the highest predicted probability among all classifiers. The logistic regression model uses the sigmoid function to map any real-valued number into the range [0, 1].

When using LDA, it took a long time. It is a time consuming method for multi-class classification.

3.4 Decision Tree

A decision tree is a supervised machine learning algorithm used for both classification and regression tasks. It works by recursively partitioning the input space into regions and assigning a label or value to each region. The decision-making process resembles a tree-like structure, where each internal node represent-s a decision based on a feature, each branch represents an outcome of that decision, and each leaf node represents the final output (class label or regression value).
 
Decision trees are advantageous because they are easy to interpret, don't require feature scaling, and can handle both numerical and categorical data. However, they are prone to overfitting, especially when they grow deep, capturing noise in the training data. 
 
3.5 Random Forest
 
Random Forest is an ensemble learning method that belongs to the family of tree-based models. It is an extension of the decision tree algorithm and is widely used for both classification and regression tasks. At each node of a decision tree, a random subset of features is considered for splitting. Random forest decorrelates the trees and reduces overfitting.

import mido

import pygame

def generate_midi(melody):
 midi = MidiFile()
 track = MidiTrack()
 midi.tracks.append(track)

 for note, duration, fermata in melody:
 # Note On message
 track.append(Message('note_on', note=note, velocity=64, time=0))

 # Note Off message after the specified duration
 if fermata:
 dur = int(duration * 140 * 2)
 else:
 dur = int(duration * 140)
 track.append(Message('note_off', note=note, velocity=64, time=dur)) # Assuming 480 ticks per beat

 return midi
