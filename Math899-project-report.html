4.             Natural language feature extraction 

 

Tokenization

Tokenization is typically the first step in NLP pipelines. It breaks down a sequence of text into smaller units called tokens. By dividing text into tokens, we can analyze and process the information at a more granular level, enabling us to derive meaningful insights and perform various linguistic operations. In my project, I use nltk.word_tokenize() to break sentences to tokens.

 

Stemming and Lemmatization

Stemming and lemmatization are two common techniques used in natural language processing (NLP) to reduce words to their base or root forms, which can help in text normalization and improve the performance of various NLP tasks. While both methods aim to achieve similar results, they differ in their approach and outcomes.

Stemming:

Stemming is a process of reducing words to their base or root forms by removing suffixes and sometimes prefixes. The resulting stems may not always be actual words or have any meaning on their own. Stemming algorithms apply a set of predefined rules to chop off common word endings, allowing different variants of a word to be represented by the same stem. For example:

• “program”, “programming,” “programs”, “programmed”, and “programmer” would all be reduced to the stem “program”.

Stemming is a simple and computationally efficient method but can result in overstemming.

Lemmatization:

Lemmatization, on the other hand, aims to determine the base form or lemma of a word while preserving its actual meaning. It produces more meaningful and interpretable results compared to stemming, but it can be computationally more expensive.

For example:

• “program”, “programming,” “programs”, and “programmed”, would be reduced to the stem “program.”  “programmer” would be remain as “programmer”.

In my project, I select lemmatization, because I want more meaningful and interpretable. And computationally efficient is less important in this project.  I used WordNetLemmatizer from nitk.stem.

5.             Model selection

 

5.1.        KNN

K-Nearest Neighbors (KNN) is a simple and intuitive supervised machine learning algorithm used for both classification and regression tasks. It is based on the principle of finding the k nearest data points in the training dataset to a given test point and making predictions. It based on most of the response or averaging the target values of those nearest neighbors. In this project I tried k=1,3,5,7,9.

 

5.2.        Multinomial Logistic Regression

Multinomial Logistic Regression is a variation of logistic regression that extends the binary classification to handle problems with multiple outputs. In binary logistic regression, we predict the probability of an event occurring or belonging to a certain class (For example 0 or 1) based on the given features. It uses a set of binary logistic regression models, one for each class, and employs a "one-vs-rest" or "one-vs-all" approach. In the "one-vs-rest" approach, for each class, the algorithm builds a binary logistic regression model that predicts the probability of that class versus the probability of all other classes combined.

 

5.3.        Naïve Bayes

Naive Bayes is based on Bayes' theorem, which relates conditional probabilities. Bayes' theorem states that the probability of an event A (posterior probability A given the evidence) given event B can be calculated using the prior probability of A, the conditional probability of B given A, over the marginal probability of B(prior probability that the evidence itself is true).

 

 

 

The assumption is that the presence or value of one feature does not affect the presence or value of any other feature.

Naive Bayes assigns the class label with the highest posterior probability as the predicted class for the new instance.

 

5.4.        Random Forest

Random Forest is a learning algorithm that combines multiple decision trees to make predictions. Random Forest uses a technique called bootstrap aggregating, or bagging, it is to create diverse training datasets. Each decision tree in the Random Forest is trained on a random subset of the original data. During the construction of each decision tree, only a random subset of features is considered at each node. This helps to introduce diversity among the trees and prevents overfitting.

 

5.5.        Boosting

Boosting algorithms combines multiple weak classifiers to a final strong classifier.

It builds weak learner sequentially. A weak learner is trained on a modified version of the training data. After added a weak learner, we readjusted the data weight. Misclassified input data gain a higher weight, and the correct classified data lose weight. So that later weak learners focus on the previous misclassified learners.

 

5.6.        XGBoosting

XGBoost (Extreme Gradient Boosting) is an optimized gradient boosting algorithm. The key differences between boosting and XGBoost: Boosting is a general concept, XGBoosting is one of the boosting. XGBoost includes several enhancements and optimizations, such as approximate tree learning, parallel and distributed computing, regularization techniques, and efficient handling of missing values.

 

5.7.        LSTM (RNN)

LSTM (Long Short-Term Memory) is one of the recurrent neural network (RNN). It is designed to overcome the limitations of traditional RNNs in capturing and retaining long-term dependencies in sequential data. Traditional feed-forward neural networks struggle to effectively capture long-term dependencies in sequential data because they lack a form of memory. LSTM introduces memory cells and gating mechanisms to solve the issue. The three key components of LSTM are: forget gate, input gate and output gate.
